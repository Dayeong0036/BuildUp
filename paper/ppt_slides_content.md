**2단계 전이학습 파이프라인**
- Phase 1: 통합 기초 학습 (SkillIntro)

- VsMelee v1 → v2: 보스 HP 상향 (누적 딜량 대비 부족 해소)
  - WR 5.9% → 59.5% (Q4)

**핵심:** 보상 설계 이전에 환경/스탯 밸런스가 학습 전제 조건

## 슬라이드 5 — 핵심 인사이트 및 문제점 (다크)

**제목:** 핵심 인사이트

**검증된 것**
- 매치업별 보상 분리 설계 → 실제 전략 분화 발생
  - VsRanged: "접근하면 이긴다" → 추격 특화
  - VsMelee: 상호 피해 수반 → 반격 전략
- 환경 설계(이동속도, 스탯)가 보상 설계보다 선행 조건

**미해결 과제**
- VsCC 보상-성과 불일치 (reward misalignment)
  - CC 스킬이 접근 자체를 봉쇄 → 평균 거리 31.8m
  - Timeout 89.0% — 보스가 시간 내 처치 불가
- VsHybrid 승률 42.6% 정체
  - 내부 지표(HR, 전투시간)는 개선 but 승리 전환 미달
- 오버트레이닝 현상 (VsMelee 추가 학습 시 WR 하락)

## 슬라이드 6 — 추후 계획 (다크)

**제목:** 추후 계획

**1. 학습 환경 ↔ 서버 환경 통합**
- 현재: 로컬 학습 전용 환경 (Unity ML-Agents)
- 목표: Host-authoritative 서버 구조와 학습 환경 통합
- 학습된 모델이 실제 멀티플레이 서버에서 직접 동작하도록 파이프라인 구축

**2. 세분화된 특화 학습 확장**
- 현재: 2명의 플레이어가 동일 빌드 → 같은 매치업으로 학습
- 목표: 각 플레이어가 서로 다른 특화 캐릭터
  - 예) Player 1 = 근접 + Player 2 = 원거리 → 혼합 매치업
  - 예) Player 1 = CC + Player 2 = 하이브리드 → 복합 대응
- 2명 × 4빌드 = 최대 10종 조합별 특화 모델

**3. VsCC 보상 구조 재설계**
- CC 내성 메커니즘 도입
- Timeout 패널티 강화
- 접근 보상 세분화

**4. 런타임 빌드 감지 → 모델 자동 전환**
- 플레이어 빌드 실시간 분류
- 매치업에 맞는 특화 모델로 동적 전환
