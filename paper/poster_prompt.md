## 1. 프로젝트 개요

### 게임 정보
- **타이틀**: 테네브리스 (Tenebris)
- **장르**: 3D 탑다운 액션 로그라이트 (2인 협동 보스전)
- **엔진**: Unity 2022 LTS
- **네트워크**: Host-authoritative (Unity Netcode for GameObjects 기반, 추후 적용)

### 핵심 주제
- **강화학습(RL) 기반 보스 AI 설계**: 수작업 행동 트리(BT) 대신 강화학습으로 보스 AI를 학습시켜, 다양한 플레이어 전략에 적응하는 동적 보스 전투 경험을 구현
- **매치업별 특화 학습**: 플레이어 빌드(근접/원거리/하이브리드/CC)에 따라 보스가 다른 전투 전략을 학습하도록 보상 함수를 분리 설계

### 연구 동기 (Introduction용)
- 전통적 보스 AI (행동 트리, FSM)는 개발자가 사전 정의한 패턴만 실행 → 플레이어가 패턴을 암기하면 난이도가 급감
- 강화학습 AI는 플레이어의 실시간 행동에 반응하여 전략을 수정 → 매번 다른 전투 경험 제공
- 특히 다양한 플레이어 빌드(근접, 원거리, CC 등)에 대해 각각 최적화된 행동을 학습할 수 있는지가 핵심 연구 질문
- 기존 RL 게임 AI 연구는 대부분 1:1 대전이나 단순 환경에서 진행 — 본 연구는 **1 vs 2 비대칭 전투** + **스킬 조합 시스템** 환경에서의 RL 적용을 탐구

## 3. 매치업별 설계 (Method용 - 핵심)

### 플레이어 프로필 시스템

플레이어 AI는 사전 정의된 행동 그래프(BehaviorGraph)로 동작하며, 빌드별로 다른 스킬풀과 이동 파라미터를 가짐.

| 프로필 | 스킬 구성 | 최적 거리 | 위험 거리 | 전투 스타일 |
|--------|-----------|-----------|-----------|-------------|
| **VsRanged** (PlayerRangedKiter) | PiercingShot(사거리66), RuptureMagazine(42), ErosionField(42), HuntingMark(66), SealChain(66) | 30~40m | 15m 이내 | 원거리 카이팅, 접근 시 도주 |
| **VsHybrid** (PlayerHybridSurvivor) | 근접+원거리 혼합 | 중거리 | 가변 | 상황에 따라 근접/원거리 전환 |
| **VsCC** (PlayerCCDebuffer) | CC(경직/침묵/감속) 위주 | 중~장거리 | 가변 | 보스 행동 억제, 지속 CC |

### 보상 함수 설계 — 17개 보상 신호

보스 에이전트는 17개의 개별 보상 신호를 가중합하여 학습합니다. 매치업마다 가중치를 다르게 설정하여 특화된 전투 전략을 유도합니다.

| 보상 신호 | 설명 | VsRanged | VsHybrid | VsCC |
|-----------|------|----------|----------|------|
| StepPenalty | 스텝당 시간 벌점 (빠른 종료 유도) | 0.0008 | 0.001 | 0.0005 |
| FireReward | 스킬 발사 보상 | 0.05 | 0.03 | 0.05 |
| MissPenalty | 스킬 빗나감 벌점 | 0.04 | 0.06 | 0.02 |
| OutOfRangePenalty | 사거리 밖 발사 벌점 | 0.15 | 0.12 | 0.08 |
| DamageReward | 대미지 비례 보상 | 0.75 | 0.6 | 0.8 |
| ShieldDamageReward | 쉴드 대미지 보상 | 0.35 | 0.3 | 0.4 |
| HitReward | 명중 보상 | 0.35 | 0.25 | 0.35 |
| Progress | 전투 진행도 보상 | 0.4 | 0.3 | 0.45 |
| Align | 타겟 방향 정렬 보상 | 0.004 | 0.004 | 0.003 |
| IdlePenalty | 무행동 벌점 | 0.01 | 0.006 | 0.012 |
| RangeMaintain | 적정 거리 유지 보상 | 0.15 | 0.15 | 0.12 |
| RangeTolerance | 거리 허용 범위 | ±1.5m | ±2.0m | ±2.5m |
| TouchReward | 최초 접촉 보상 | 0.3 | 0.2 | 0.3 |
| PlayerKilledReward | 플레이어 처치 보상 | 0.5 | 0.4 | 0.6 |
| CounterAttack | 반격 보상 (플레이어 캐스팅 중 공격) | 0.06 | 0.08 | 0.10 |
| CloseAlignBonus | 근접 시 정렬 추가 보상 | 0.002 | 0.003 | 0.002 |
| RetreatPenalty | 접촉 후 후퇴 벌점 | 0.005 | 0.003 | 0.004 |

**VsRanged 핵심 설계 의도:**
- DamageReward(0.75)와 HitReward(0.35)를 높여 공격적 전투 유도
- RangeMaintain(0.15)으로 원거리 플레이어 추격 시 적정 거리 유지
- FireReward(0.05) > MissPenalty(0.04) → 사거리 내에서는 적극적 발사 유도 (net +0.01)
- OutOfRangePenalty(0.15) 높게 설정 → 사거리 밖 무의미한 발사 억제

### 학습 파이프라인

```
Phase 1: SkillIntro (통합 학습)
  └─ 기본 이동 + 스킬 사용법 학습 (모든 매치업 혼합)
  └─ checkpoint.pt 생성

Phase 2: 매치업 특화 학습 (Transfer Learning)
  ├─ VsRanged: init_path=SkillIntro/checkpoint.pt → 원거리 특화
  ├─ VsHybrid: init_path=SkillIntro/checkpoint.pt → 하이브리드 특화
  └─ VsCC: init_path=SkillIntro/checkpoint.pt → CC 특화

런타임: RewardPreset enum으로 매치업 선택 → 해당 가중치 자동 적용
```

### 핵심 발견사항

1. **매치업별 특화 학습의 유효성 확인**: 동일한 RL 프레임워크에서 보상 가중치만 변경하여 전혀 다른 전투 전략을 학습시킬 수 있음을 실증
2. **V자 학습 곡선**: PPO 알고리즘의 특성인 탐색-수렴 패턴이 VsRanged에서 명확히 관찰됨 (Q2-Q3 탐색 딥 → Q4 수렴)
3. **환경 설계의 중요성**: VsRanged v1 실패(WR 0.6%) → v2 성공(WR 84.1%)은 보상 함수만으로는 부족하고, 물리적 환경(속도, 대미지 밸런스)이 학습 가능성을 결정함을 보여줌
4. **보상-승률 괴리**: VsCC에서 Reward는 지속 상승하지만 WR은 정체 → 보상 함수 설계와 실제 게임 목표 간 정렬(alignment)이 RL 성공의 핵심 과제
5. **CC 메커니즘의 RL 학습 난이도**: 행동 억제(경직/침묵)는 에이전트의 탐색 자체를 방해하여 학습을 구조적으로 어렵게 만듦

1. **VsCC 보상 구조 재설계**: CC 상태에서의 반격 보상 강화, CC 내성 메커니즘 도입, Timeout 패널티 재조정

7. **실제 플레이어 테스트**: RL 학습된 보스 AI vs 실제 플레이어의 체감 난이도 및 재미 요소 평가

## 8. 참고 키워드 / 용어 정리

| 용어 | 설명 |
|------|------|
| PPO (Proximal Policy Optimization) | OpenAI가 제안한 정책 경사 기반 강화학습 알고리즘. 클리핑을 통해 안정적 학습 |
| ML-Agents | Unity에서 강화학습 에이전트를 학습시키기 위한 오픈소스 프레임워크 |
| Transfer Learning | 사전 학습된 모델의 가중치를 초기값으로 사용하여 새로운 태스크에 빠르게 적응 |
| Reward Shaping | 최종 보상만으로는 학습이 어려운 경우, 중간 보상 신호를 설계하여 학습을 유도 |
| Exploration-Exploitation | 새로운 전략 탐색(exploration)과 기존 최적 전략 활용(exploitation) 간의 균형 |
| V-shaped Learning Curve | PPO에서 자주 관찰되는 패턴. 탐색 단계에서 일시적 성능 하락 후 수렴 |
| Observation Space | 에이전트가 환경으로부터 관측하는 정보의 차원과 구조 |
| Action Space | 에이전트가 취할 수 있는 행동의 종류와 구조 |
| Reward-WinRate Misalignment | 보상 함수 값은 증가하지만 실제 승률은 증가하지 않는 현상 |
| CC (Crowd Control) | 경직, 침묵, 감속 등 상대방의 행동을 제한하는 스킬 유형 |
