# Codex 작업 요청: SkillIntro 통합 학습을 baseline으로 활용하는 방안 검증

## 배경

현재 논문의 핵심 주장 중 하나는 "매치업별 보상 분리 설계가 유효하다"이다.
그런데 현재 초안에는 **비교 대상(baseline)**이 명시되어 있지 않다.
심사위원이 "분리 안 해도 그 정도 나오는 거 아닌가?"라고 질문하면 답이 없는 상태이다.

## 제안

Phase 1 SkillIntro(통합 학습)의 성능 수치를 baseline으로 활용한다.

### 논리 구조

```
SkillIntro(통합 보상, 4종 혼합) → 매치업 무관한 범용 정책
    ↓ 전이학습
Phase 2(매치업별 보상 프리셋) → 매치업 특화 정책
```

비교: SkillIntro 체크포인트 시점의 각 매치업별 성능 vs Phase 2 특화 학습 후 성능

### 이것이 baseline으로 적합한 이유

1. 동일한 네트워크 구조(128×2, PPO)를 사용
2. 동일한 관측/행동 공간
3. 차이점은 오직 보상 가중치뿐 → 보상 분리 설계의 효과를 분리 측정 가능
4. 이미 Phase 1 학습이 완료되어 있어 추가 실험 불필요
5. Phase 2의 init_path가 SkillIntro 체크포인트이므로, Phase 2 Q1이 사실상 SkillIntro 정책의 성능에 가까움

### 사용 가능한 데이터

SkillIntro 통합 학습 결과는 `results/SkillIntro_Comprehensive/` 에 있으며,
각 매치업별 CSV 파일의 초기 구간(Q1)이 전이학습 직후 = SkillIntro baseline에 해당한다.

Phase 2 각 매치업의 Q1 vs Q4 비교:

| 매치업 | Q1 WR (≈baseline) | Q4 WR (특화 후) | 개선폭 |
|--------|-------------------|----------------|--------|
| VsRanged | 84.1% | 84.1% | 0%p (이미 수렴) |
| VsMelee | 54.2% | 59.5% | +5.3%p |
| VsHybrid | 38.1% | 42.6% | +4.5%p |
| VsCC | 11.3% | 10.3% | -1.0%p (개선 실패) |

### 해석 의도

- VsRanged: 환경 조정이 핵심이었고, 보상 특화의 추가 효과는 V자 탐색-수렴 패턴으로 나타남
- VsMelee: 특화 학습으로 +5.3%p 개선, 보상-승률 정렬 양호
- VsHybrid: +4.5%p 개선, 내부 지표(HR +15.3%p, Duration -30%)는 더 큰 개선
- VsCC: 특화 학습으로도 개선 실패 → 보상 구조 자체의 한계 확인

## 검토 요청

1. SkillIntro Q1을 baseline으로 사용하는 것이 학술적으로 타당한가?
   - "Q1 = 전이학습 직후 = 통합 정책 성능"이라는 가정이 합리적인가?
   - 엄밀하게는 Q1도 이미 특화 보상으로 일부 학습된 상태인데, 이 오차는 허용 가능한가?

2. 방안 A/B/C 중 어떤 것이 학부 학회 논문에 가장 적합한가?

3. VsRanged Q1=Q4=84.1%인 경우, "특화 효과 없음"이 아니라 "V자 탐색 후 동일 수준 회복"으로 해석해야 하는데, 이 뉘앙스를 baseline 비교에서 어떻게 표현하면 좋은가?

4. VsCC Q1(11.3%) → Q4(10.3%)로 오히려 하락한 점은 "특화 학습이 역효과"인가, "통합 정책도 이미 한계"인가?

5. 추가 의견
