## 수정 1: graph5_vscc_reward_wr.png 내부 주석 완곡화

### 현재 상태
- 그래프 안에 "보상 증가 ≠ 승률 개선" 박스 텍스트
- 그 아래 "Reward Misspecification" 이탤릭 텍스트

### 문제
- "Reward Misspecification"은 Pan et al.(2022)의 정의에서 "보상 함수가 설계자의 의도와 체계적으로 어긋나는 상황"을 뜻함
- 본 실험은 4종 매치업 중 1종(VsCC)에서만 이 현상이 관찰되었고, 원인 분석도 "CC가 접근을 방해하여 보상 획득 경로와 승리 경로가 분리됨" 수준의 가설 단계임
- 학부생 논문에서 확정적 용어로 쓰면 심사위원이 "그럼 어떤 보상이 misspecified인지 분리 실험은 했는가?" 같은 질문을 던질 수 있음
- 따라서 그래프 내부 텍스트도 "가능성 시사" 수준으로 낮추는 것이 안전

### 요청 수정안
- "보상 증가 ≠ 승률 개선" → "보상 ↑ 승률 정체" (간결화)
- "Reward Misspecification" → "보상-성과 불일치 관찰" (한글 + 완곡 표현)

### 수정 위치
`poster_graphs/generate_all.py` 154~178행 부근, Graph 5 섹션의 ax1.annotate() 두 줄

## 수정 4: 본문 내 박스형 주석 축소 검토

### 대상
generate_all.py 내 그래프별 annotate() 텍스트 중 논문용으로 과한 것들:
- graph2: "탐색 구간" 이탤릭 주석 → 논문에서는 캡션으로 설명 가능, 그래프 내 주석은 유지해도 무방
- graph4: "HR +15.3%p", "전투 시간 약 30% 감소" → 이 그래프가 본문에서 빠질 경우 수정 불필요
- graph8: "WR 60% 부근 수렴 / 보상은 지속 개선" → 유지 가능하나 폰트 크기가 크면 축소 고려

### 판단 기준
- 포스터용이면 주석 밀도 현재 수준 OK
- 논문용이면 데이터 라벨 + 캡션 설명으로 충분하고, 해석성 주석("탐색 구간", "약 30% 감소")은 줄이는 것이 깔끔
- 본 논문은 학회 논문집(2~5페이지)이므로 중간 수준이 적절

### 요청
- graph5 내부 주석은 반드시 수정 (수정 1)
- 나머지는 Codex 판단에 맡김: 유지/축소/제거 중 추천안 제시
